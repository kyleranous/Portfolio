Home Automation

Setting up your own home automation server

My current home server (1x Dell R710) is starting to get pretty full and I want to free up some resources. I figure the best way is to move my home automation server onto its own setup. I have some requirements for my new setup:

  • The setup must be cheap - preferably using components I have lying around, or that I can pick up cheap and fast
  • It must be reliable – Automated Backups, and self healing as much as possible.
  • It should be scalable – If I need more resources it should be quick and cheap to add resources.
  • It should increase my skillset – I want to learn something new doing this.
  • It should be fun – I love automation and this opportunity to refresh my setup should be fun to do, and at least not painful to maintain.

Looking at my requirements, a cluster of raspberry pi’s sounds like a great idea. I have a stack of pi 3B and 3B+ from old projects, and the 3B and 3B+ can be picked up on amazon for less then $40. A docker swarm will redeploy containers if a node goes offline, and with some additional configuration the swam can auto-scale microservices that require more resources, as needed. PLUS I have never built a cluster before so I get to learn something new.

I am breaking this project into smaller chunks. At the end of each entry, there will be a functioning section of the project that can be used and grown upon (Home Assistant is installed in part 3). Any code that is written will be published on my github. You can replicate the code yourself or just take and run my code. After the server is setup and running we will head into my shop to build an enclosure for the server and a control panel, then we will start looking at designing our own IoT devices.
