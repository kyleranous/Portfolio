HANA

Home Automation Network Assistant

Creation of a swarm and container orchastration for running microservices can be incredibly handy when setting up automation systems. However, container and swarm management can have a steep learning curve that may not be accessable to home automation enthusiasts or small businesses.

Why use a swarm for home automation?

While it may seem like setting up a swarm for home automation is a little overkill, there are amazing benifits that a swarm can provide. A common complaint about commercial home automation systems is the dependency on connection to an outside resource. "Sorry, Internet is down, I can't change my thermostat settings." However, if we self-host a home automation system on a single machine, and the breaker trips, or the NIC fails, or we have to run updates on the machine that require restarts, we are in the same exact boat. A Swarm or cluster setup lets us use nodes for redundency. Setting the program up to run as a service allows it to be automatically redeployed to another node if something happens to the one it is running on. If you have family in town and your Kodi or Plex server is running low on resources, it can be quickly scaled to more machines and traffic automatically routed, If your service managing your thermostat goes down in the middle of the night or while you are gone in the winter, it will be automatically redeployed and you will come home to a node that needs to be fixed, but no burst pipes.

What is HANA?

HANA is a server management tool with the goal of making the setup and management of Docker swarms easier. It is built with Django and uses the Docker API and Python SDK to communicate and send commands to the swarm. HANA has both a browser based UI and an API to assist integration with other automation programs like Home Assistant and Node-Red.

Frameworks/Packages Used:

  • Django 4.0.3
  • Django Rest 3.13.1
  • Bootstrap 5
  • drf-yasg 1.20

Features

  • Add existing swarms to be tracked and managed by HANA - HANA can track multiple swarms.
  • Swarm Dashboard that displays quick glance Node and Service Data
  • Node Detail pages that display resource utilization, containers on the node, and quick actions like changing the state of the node (Active, Pause, Drain), or Promoting/Demoting a node. Useful when a node needs to be taken off-line for maintenance.
  • Service Detail Pages that allow quick actions like pausing/restarting a service, and scaling services up and down.

NOTE:
HANA requires the nodes in the swarms to expose their ports and depending on configuration will run commands as a root user. Currently there is no support for TLS or certificates or authentication on API endpoints (On Development Roadmap). If you are using or testing Hana in it's current state, DO NOT expose it outside of your network.

Planned Features

  • Guided Swarm Setup with script generation to speed up and maintain consistancy when creating new swarms.
  • TLS support to secure the connection between HANA and your swarms.
  • Authentication requirements on API Endpoints
  • Single Click Image Deployments to quickly deploy commonly used resources on a swarm
  • Docker Compose file creation and saving to create your own single click deployments
  • Service Navigation Dashboard, A single page that will contain links to running services that have a web ui.
  • Swarm Health Checks - Check the status of a swarm and make basic reccomendations
  • In Browser Terminal Emulators and SSH client for working directly on nodes or in containers.
  • Guided Distributed Volume creation and management. To assist in creating redundant storage that can be used by your services.
  • Guided Private Container Registry creation and management so you can host the containers you want to use on your network, or add your own containers.
  • Swarm and Service Documentation generation.
